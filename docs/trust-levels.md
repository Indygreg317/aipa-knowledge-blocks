# Trust Levels

Trust levels define the confidence in a Knowledge Block based on review, testing, and operational history.

## Purpose

To classify how much confidence can be placed in a Knowledge Block before execution.

Trust is not static. It reflects past validation, not current conditions.

## Levels

### Draft
- Newly created
- Not reviewed
- Not safe for execution

### Reviewed
- Evaluated by domain experts
- Logic and structure validated
- Limited confidence

### Verified
- Tested in controlled scenarios
- Evidence supports expected outcomes
- Higher confidence

### Production-Proven
- Successfully executed in real environments
- Demonstrated reliability over time
- Highest confidence level

## Important Distinction

Trust is based on history.

Execution is based on current conditions.

A highly trusted Knowledge Block can still be unsafe if conditions have changed.

## Key Principle

Trust influences execution decisions.

It does not guarantee execution approval.

## Relationship to Execution Control

Execution control uses trust as one input, along with:

- current context
- policy state
- risk posture
- authority

## Outcome

Trust levels provide context.

Execution control provides enforcement.
